Mobile Overview

Web visitors are increasingly using a wide variety of devices other than a desktop computer to access the Internet. Websites need to be able to accomodate as many of these devices as possible to avoid "leaving out" a increasing number of visitors and potential customers.

We consider mobile usage to be devices that are intended to be portable and generally use smalller screens and input devices that are not desktop mice.

Portable use can also imply a lower bandwith usage pattern and lack of helper plugins such as Flash.

  • Content must be readable on smaller screens. - Even though modern smartphones and tablets feature high resolution screens, there still exists a minimum size under which text and images become very difficult to read and decypher.
  • Where screen "real estate" is limited by screen size, priority should be given to UI elements that deliver the primary content message and UI that serves a secondary purpose should be moved to lesser positions, reduced or removed completely.
  • Accomodation for touchscreen interfaces should be given. Action targets must be large enough to easily actuate with a touch while avoiding activating unwanted controls. Actions triggered by mouse hover should have a touch action as a secondary interatction.
  • There is no "one" size for mobile screens, so some care must be taken to allow the user interface to adapt to a vide range of screen sizes and orientations.
  • Lower bandwith duting mobile use should be considered and pages must be optimized to deliver content in a short time. Portable use can imply that a visitor is busy with another task and the speed at which she can obtain the desired information from the website is extremely important.

Approaches

Mobile-specific Site

This approach calls for a separate side design specifically for mobile visitors.

This is becoming less common due to the larger costs to develop a standalone site and as responsive techniques become more advanced.

Mobile First

The mobile first approach implies that the mobile UX is designed first and desktop-friendly enhancements are added on top of this. The advantage is mobile development is not shortchanged by trying to adapt a desktop UI.

This method is also considered Progressive Enhancement which means that the basic UI is available to all clients and additional features are accessible to clients that support them.

Mobile Responsive

This is the basic term to describe a UI that is designed to adapt to differeent screen sizes by allowing page elements to shrink, zoom, change form or disappear in a smooth way as the screen size is changed.

This is very similar to the mobile first approach with the main difference in the way the UI features are planned. In the responsive approach, equal weight is given to desktop and mobile visitors.

Retina Displays

A "retina" display is a screen that is generally twice the standard display resolution or more. Type looks much sharper and detailed on a retina screen and images can look worse because pixelation and compression artifacts can be visible that were not noticable at standard resolution.

To improve the experience for retina display visitors, we can supply a double resolution image to those devices and the normal resolution image to all other devices.

The double resolution assets will need to be specified as a deliverable as upscaling a normal resolution image will not produce the desired quality.

Touch Interaction

Some special consideration need to be given to ensure that the UI is usable via touchscreen. Touch targets need to be larger and have more safe space around them.

Mouse hover is not an option for touch screen devices, so all interations that would normally be activated on hover need a clickable toggle interaction as well.

Dragging objects on a touchscreen is more awkward than with a mouse, so a different interaction is recommended.

Touch screen devices have some additional interation modes like pinching and easier horizontal scrolling that are not present with a mouse, so some consideration for advanced interations could be explored.
